# First Week Setup for a New Company

![New company drawer showing account fields and accounting edit permissions](../screenshots/company-setup-and-migration/company-account-fields-defaults-step-01.png)

Use this first-week setup workflow to move from a new company record to a usable accounting file with core defaults, lists, bank accounts, opening balances, and review reports.

## When To Use This

Use this page when you have just created or imported a company and want a practical setup path before day-to-day bookkeeping begins.

## Before You Start

- You know whether the company is starting fresh or being migrated from another accounting system.
- You know the preferred chart of accounts approach.
- You know the bank and credit card accounts that need to be tracked.
- You know whether the company needs invoices, bills, checks, or all three.

## Steps

1. Create the company or import the company data.
2. Confirm the active company in the sidebar before continuing.
3. Review company-level setup fields.
   - Confirm company name, country, currency, fiscal year, and posting cutoff date.
   - Confirm required account fields and account display preferences.
4. Review the `Chart of Accounts`.
   - Confirm cash, bank, credit card, receivable, payable, income, expense, and equity accounts.
   - Add or clean up accounts before entering daily activity.
   - Confirm default AR and AP accounts if you plan to use invoices and bills.
5. Set up customers before entering invoices.
   - Review payment terms.
   - Add customer records needed for open invoices or new billing.
6. Set up vendors before entering bills or checks.
   - Add vendor records needed for open bills or common payments.
   - Add default expense accounts where repeated vendor coding will help.
7. Set up items before invoicing if the company uses item or SKU-based invoice lines.
8. Add or review bank and credit card accounts.
9. Enter opening balances only when you understand the opening date and account balances being carried forward.
10. Import or enter open AR and AP items if the company is moving from another system.
11. Run setup review reports.
   - `Trial Balance`
   - `Balance Sheet`
   - `Income Statement`, if there is beginning-period activity
   - Aging reports, if open AR or AP is entered
12. Review one sample workflow before broad entry begins.
   - Create a test invoice path if sales will be used.
   - Create a test bill or check path if payables will be used.
   - Import and confirm a small set of banking activity if bank feeds/imports are part of the process.

## What Happens Next

The company should be ready for normal bookkeeping with fewer surprises around missing accounts, default AR/AP settings, uncategorized bank activity, or incomplete customer/vendor lists.

## If Something Looks Wrong

- Do not enter current-month transactions until the active company is confirmed.
- Do not import source data into a company that still has the wrong account setup.
- Do not skip default AR/AP account review if invoices or bills will be used.
- Do not create opening balances without a clear cutoff date.
- Do not use a journal entry when a source workflow is the better home for the transaction.
